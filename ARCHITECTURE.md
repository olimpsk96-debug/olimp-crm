## 0. Стратегическая позиция

### 0.1 Главный вывод из интервью

Это **директорская система**, а не учётная.

Главный пользователь — Дмитрий, директор. Главные боли (по приоритету):
1. **Тендеры/продажи** — упускаем заявки, нет воронки → кровь компании
2. **Учёт по проектам** — невидимая маржа → не понимаем, на чём зарабатываем
3. **Закрытие КС-2/КС-3** — хаос → задержки оплат, кассовые разрывы
4. Субподрядчики — контроль и оплаты
5. ОТ/ТБ — допуски, штрафы

### 0.2 Архитектурная идеология

> **«Директор открывает систему 5 раз в день и за 30 секунд понимает: что горит, что стоит денег, что требует решения сегодня».**

Всё остальное — учёт, документы, бухгалтерия — должно работать **в фоне**, генерируя данные для этих 30 секунд. Прорабы и сметчики обслуживают директорский уровень, а не наоборот.

| ❌ **Полный кастомный фронт** | Слишком долго, теряем обновления ERPNext |

## 4. Кастомное приложение `olimp_construction`

В терминах Frappe мы создадим **отдельное приложение**, не трогая ERPNext core. Это даст:
- Чистые обновления ERPNext без конфликтов
- Возможность переиспользовать в других ваших компаниях / партнёрам
- Лёгкое тестирование

## 5. Дизайн-система (Apple-style для ERP)

### 5.1 Принципы

1. **Negative space** — много воздуха между элементами
2. **Typography как иерархия** — Fraunces (Display) + Inter Tight (Body) + JetBrains Mono (числа)
3. **Тёмная тема по умолчанию** — для длительного использования директором
4. **Числа — главный персонаж** — `font-feature-settings: 'tnum'` всегда
5. **Анимации только содержательные** — entrance, transitions, не декорация

### 5.2 Палитра

```css
/* Базовые тёмные тона */
--bg-base:       #0A0A0B;  /* фон страницы */
--bg-elevated:   #131316;  /* шапки, sidebar */
--bg-card:       #18181C;  /* карточки */
--bg-card-hover: #1E1E23;  /* hover-состояние */

/* Границы */
--border-subtle: rgba(255, 255, 255, 0.06);
--border-strong: rgba(255, 255, 255, 0.12);

/* Текст */
--text-primary:   #F4F4F5;  /* заголовки и акценты */
--text-secondary: #A1A1AA;  /* основной текст */
--text-tertiary:  #71717A;  /* подсказки, метки */

/* Семантика */
--accent:   #F97316;  /* фирменный — оранжевый, индустриальный */
--success:  #34D399;  /* зелёный — деньги пришли, KS-2 подписан */
--warning:  #FBBF24;  /* жёлтый — внимание, дедлайны */
--danger:   #F87171;  /* красный — просрочки, риски */
--info:     #818CF8;  /* индиго — нейтральные события */
```

### 5.3 Типографика

| Шрифт | Использование |
|-------|---------------|
| **Fraunces** (Display, weights 300-600) | Заголовки H1/H2, "красивые" моменты — приветствие на дашборде, названия объектов |
| **Inter Tight** | Весь body-текст, навигация, кнопки |
| **JetBrains Mono** | Все числа, ID, даты, мониторинговые маркеры |

Отказались от Inter (банально) и SF Pro (закрытый, не для веба). **Fraunces** — серифный, тёплый, не как у всех. **Inter Tight** — компактнее обычного Inter, лучше для плотных дашбордов.

### 5.4 Сетка и spacing

- Базовая единица: `4px`
- Радиусы: `8px` (мелочи), `12px` (карточки), `16px` (большие карточки)
- Тени: предельно мягкие (`0 20px 40px -20px rgba(0,0,0,0.8)`)
- Анимации: `cubic-bezier(0.16, 1, 0.3, 1)` — Apple ease-out

### 5.5 Иконография

- Только outline, толщина `1.6px`
- Размеры: 13/14/16/18/20px
- Custom SVG (без библиотек) для главных, Lucide React для второстепенных

## 6. Бэкенд: REST API для Next.js

ERPNext имеет встроенный REST. Создаём **дополнительный слой** для дашбордов:

```python
# olimp_construction/api/dashboard.py
@frappe.whitelist()
def get_director_dashboard():
    return {
        "kpi": {
            "revenue_ytd": ...,
            "margin_pct": ...,
            "cashflow_forecast": ...,
            "tenders_active": ...,
        },
        "active_projects": [...],     # 7 объектов
        "hot_tenders": [...],         # топ-3 горящих
        "ot_alerts": [...],           # истекающие допуски
        "activity_feed": [...],       # лента событий
    }
```

**Все эти запросы кэшируются в Redis на 60 сек** — дашборд должен открываться за <300мс.

## 8. Безопасность

| Уровень | Меры |
|---------|------|
| **Сеть** | Cloudflare (DDoS), fail2ban на SSH, UFW firewall |
| **Доступ** | SSH только по ключам, 2FA для всех users в ERPNext |
| **Данные** | Backup в Yandex Object Storage (ежедневно), encryption-at-rest |
| **Компрометация** | Audit log в ERPNext (стандарт), мониторинг в n8n |
| **VPN** | Опционально: WireGuard, чтобы admin-панель была доступна только через VPN |

**Важно:** учитывая твоё внимание к давлению ИФНС — данные хранятся **только в РФ**, бэкапы тоже в РФ.

## 9. Принятые архитектурные решения (ADR)

| ID | Решение | Обоснование |
|----|---------|-------------|
| ADR-1 | Гибрид (ERPNext + кастомный Next.js фронт) | Полный кастом — 6+ мес. Сток — не Apple-style. Гибрид даёт оба |
| ADR-2 | Tенденрные площадки через TenderOK API | Парсинг zakupki.gov.ru — 200+ часов разработки + капчи |
| ADR-3 | Selectel вместо Beget | OpenVZ Beget блокирует Docker-фичи |
| ADR-4 | Отказ от Twenty CRM | Сырой v0.3х. ERPNext CRM хватает |
| ADR-5 | Тёмная тема по умолчанию | Главный пользователь — директор, длительные сессии |
| ADR-6 | Fraunces + Inter Tight + JetBrains Mono | Уход от ИИ-клише (только Inter), отличаемость |
| ADR-7 | n8n как hub автоматизации | Self-hosted, vendor-independent, ты уже используешь |
| ADR-8 | Custom Frappe app `olimp_construction` | Чистые upgrade, переиспользуемость, тестируемость |
| ADR-9 | Кэширование дашбордов в Redis 60с | Дашборд открывается <300мс, нагрузка на БД снижается |
| ADR-10 | PWA вместо нативного app для прорабов | Проще обновлять, не нужны магазины, работает offline |

## 10. Что **НЕ** в первом релизе

Чтобы не тонуть в scope creep, явно выводим за рамки:

- ❌ Электронная подпись документов (КЭП интеграция)
- ❌ Импорт из 1С Бухгалтерии (фаза 3)
- ❌ Парсинг тендерных площадок (используем TenderOK)
- ❌ Интеграция с Гранд-Сметой (импорт через Excel)
- ❌ Внешний портал заказчиков (фаза 5+)
- ❌ Мобильное приложение в App Store / Google Play (PWA достаточно)
- ❌ Биометрия / распознавание лиц на стройплощадке
- ❌ Интеграция с госуслугами / СБИС / Контур.ЭДО
