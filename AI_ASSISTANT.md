## 0. Зачем это нужно

Дима — директор. Каждый день у него типичные вопросы:
- «Сколько мы заработали на АКЗ за прошлый год?»
- «На каком объекте мы делали пожарную защиту в 2023?»
- «Какая средняя маржа по кровельным работам?»
- «Кто чаще всего обходит нас на тендерах за последние 6 месяцев?»
- «На каком объекте у нас был перерасход материалов?»
- «Когда последний раз была поверка крана?»

**Сегодня** ответы на эти вопросы:
- 5-30 минут поиска по Excel-файлам / 1С / письмам
- Многие ответы Дима **не получает вообще** — забывает, или не знает, где смотреть
- Решения принимаются на интуиции, а не на данных

**Цель AI-ассистента:** ответ за 8 секунд на основе всех данных компании.

## 1. Что умеет ассистент

### 1.1 Аналитические запросы (RAG по данным)

Примеры:
> **Дима:** «Сколько объектов мы сделали для EVRAZ за последние 3 года?»
> **AI:** «Для EVRAZ с 2023 по 2025 завершено 6 объектов на общую сумму 87,4 млн ₽. Средняя маржа 38,2%. Подробности по каждому...»

> **Дима:** «На каких работах у нас лучшая маржа?»
> **AI:** «По данным за 2025: промальп — 41,3%, кровля — 39,8%, АКЗ — 36,5%. Самая низкая — усиление углеволокном (29,1%) из-за высокой стоимости материала.»

### 1.2 Контекстные запросы (про конкретный объект)

> **Дима** *(на странице Marins Park)*: «Что у нас сейчас здесь происходит?»
> **AI:** «Объект Marins Park: готовность 84%, опережение +5 дней. Сейчас в работе этап 3 (утеплитель ROCKWOOL, прогресс 65%). КС-2 №3 готов к закрытию — это +2,1 млн ₽ к cashflow. Рекомендую закрыть сегодня.»

### 1.3 Поисковые запросы

> **Дима:** «Найди договор с НПП Старт по трубам»
> **AI:** «Найдено: Договор №2024-1147 от 12.03.2024 на ремонт трубы 21А (3,4 млн ₽). Подписан, оплачен 28.04. Также есть тендерная заявка от 09.05.2026 на трубы 21А/21Б (текущий статус: подготовка КП).»

### 1.4 Прогнозные запросы

> **Дима:** «Когда у нас будет кассовый разрыв?»
> **AI:** «Прогноз на 16.05 показывает разрыв -2,1 млн ₽. Главные причины: задержка платежа от Эдельвейса (медиана оплаты у них 28 дней, последняя КС-2 подписана 12.05). Рекомендую: (1) закрыть КС-2 №3 на Marins Park — +2,1 млн к 20.05, (2) напомнить Эдельвейсу.»

### 1.5 Управленческие запросы

> **Дима:** «Какие тендеры стоит подавать?»
> **AI:** «Сейчас в воронке 4 тендера в статусе "Найдено". По AI-анализу подходят:
> 1. Промальп. работы EVRAZ (match 89%, есть опыт, 6,1 млн)
> 2. Кровля Газпром Уренгой (match 81%, профиль наш, 14,7 млн)
> Остальные 2 — низкий приоритет (match < 65%).»

### 1.6 Операционные действия

> **Дима:** «Создай КС-2 на этап 3 Marins Park»
> **AI:** «Подготовлю КС-2 №3 на 2,1 млн ₽ на основе фактического выполнения этапа 3 (утеплитель + стяжка). [Открыть превью] [Создать]»

### 1.7 Голосовые запросы (на ходу)

> **Дима** *(в машине, голосом)*: «Слушай, что у нас по тендерам сегодня горит?»
> **AI** *(голосом или в Telegram)*: «Один тендер с дедлайном через 2 дня — АКЗ ТЭЦ-3 от Т Плюс на 8,2 млн. Match score 64%, чек-лист готов на 60%. Сметчик Наталья работает.»

## 2. Что НЕ умеет (сознательно)

- Не предсказывает погоду или рынок
- Не пишет договоры с нуля
- Не принимает решения за директора (только подсказывает)
- Не делает финансовые транзакции без подтверждения
- Не отправляет письма заказчикам без явного approve
- Не выдаёт цифры, в которых не уверен (говорит «недостаточно данных»)
- Не трогает данные, к которым у пользователя нет доступа

## 3. Архитектура

### 3.1 Стек

| Компонент | Технология | Стоимость |
|-----------|------------|-----------|
| LLM (генерация) | Claude Opus 4.7 (основной), GPT-4o (бэкап) | ~15-20K ₽/мес |
| Embeddings | OpenAI text-embedding-3-large (3072 dim) | ~3-5K ₽/мес |
| Vector DB | Qdrant (self-hosted Docker) | 0 ₽ |
| Reranker | GPT-4o-mini | ~1-2K ₽/мес |
| Speech-to-text | OpenAI Whisper | ~1-2K ₽/мес |
| Text-to-speech | OpenAI TTS (для голосового ответа) | ~1K ₽/мес |
| Backend | Python (Frappe app) | — |
| Frontend | Next.js 15 + Server-Sent Events для streaming | — |

### 3.2 Поток данных (RAG flow)

```
1. Пользователь                  →   "Сколько мы заработали на АКЗ за 2025?"
                                     +
                                     контекст (user, role, current page)
                                     ↓
2. Voice (если голос)           →   Whisper API → текст
                                     ↓
3. Query Understanding           →   Claude парсит:
                                     - intent: analytics
                                     - entities: {work_type: "АКЗ", year: 2025}
                                     - filters: based on user permissions
                                     ↓
4. Embedding                     →   OpenAI: query → vector[3072]
                                     ↓
5. Hybrid Search в Qdrant       →   Vector similarity + keyword filter
                                     + access_level filter
                                     → top 20 chunks
                                     ↓
6. Reranking                     →   GPT-4o-mini: 20 → top 5 most relevant
                                     ↓
7. Context Building              →   System prompt + retrieved chunks +
                                     conversation history (last 4 messages)
                                     ↓
8. LLM Generation                →   Claude Opus 4.7 (streaming)
                                     ↓
9. Response                      →   Текст с цитированием источников
                                     + suggested actions
                                     ↓
10. Audit & Cost Tracking        →   Save tokens used + cost
                                     ↓
11. UI (streaming)               →   SSE → пользователь видит ответ по словам
```

## 5. System Prompt (стратегический)

```
Ты — AI-ассистент строительной компании ООО «Олимп» (Екатеринбург).

КОНТЕКСТ КОМПАНИИ:
- Промышленное строительство и ремонт: АКЗ, кровля, промальп, монолит, усиление углеволокном, монтаж м/к
- Оборот ~120 млн ₽/год, средняя маржа ~37%
- Основные заказчики: EVRAZ, ВИЗ-Сталь, RUSAL, РЖД, Газпром, НОВАТЭК
- Команда: директор Дмитрий Клочков, главный инженер, сметчик, ПТО, закупки, 3 прораба, бухгалтер

КАК ОТВЕЧАТЬ:
1. Используй ТОЛЬКО факты из предоставленных документов. Не выдумывай цифр.
2. Если данных недостаточно — честно скажи об этом.
3. Указывай источники: «по смете EST-2026-0078», «по договору №204746».
4. Числа — точные, в рублях, с разделителями: «8 900 000 ₽» а не «8,9M».
5. Сроки — конкретные даты в формате ДД.ММ.ГГГГ или относительные («через 5 дней»).
6. Тон: профессиональный, без эмодзи, без лишних слов.
7. Длина: краткий ответ + детали по запросу. Большинство вопросов — < 200 слов.
8. Если можешь предложить конкретное действие — предложи кнопку.

КОГДА НЕ ОТВЕЧАТЬ:
- Если запрос требует данных, к которым у пользователя нет доступа → объясни вежливо
- Если данных в базе знаний нет → скажи прямо «нет данных» и предложи добавить
- Если вопрос вне строительной тематики → коротко переадресуй

ФОРМАТЫ ОТВЕТОВ:
- Аналитика → таблица + текстовое объяснение
- Сравнение → markdown-таблица
- Прогноз → числа с диапазоном (с диапазоном уверенности)
- Рекомендация → 1-2 предложения + кнопка действия

ТЕКУЩАЯ ДАТА: {today}
ПОЛЬЗОВАТЕЛЬ: {user.name} ({user.role})
УРОВЕНЬ ДОСТУПА: {user.access_level}
```

## 6. Управление стоимостью

AI — единственная статья расхода, которая может **легко улететь в небо**.

### 6.1 Бюджет на AI

```yaml
monthly_budget:
  total: 25 000 ₽
  breakdown:
    claude_opus: 15 000 ₽      # ~3-4M tokens output
    embeddings: 3 000 ₽         # ~5M tokens
    whisper: 2 000 ₽            # ~50 hours audio
    gpt4o_mini_rerank: 2 000 ₽  # ~10M tokens
    safety_buffer: 3 000 ₽
```

### 6.2 Лимиты на пользователя

```python
PER_USER_DAILY_LIMITS = {
    "Director": {
        "messages": 100,
        "max_cost_rub": 500,
    },
    "Manager": {
        "messages": 50,
        "max_cost_rub": 200,
    },
    "Foreman": {
        "messages": 30,  # короткие, в основном через voice
        "max_cost_rub": 100,
    }
}
```

### 6.3 Оптимизации стоимости

1. **Кэширование контекста** — Claude prompt caching (90% экономии на system prompt)
2. **Embeddings один раз** — не пересчитываем для одних и тех же документов
3. **Сначала маленькая модель** — простые вопросы → GPT-4o-mini (10× дешевле)
4. **Лимит истории** — последние 4 сообщения, не вся история
5. **Rate limiting** — 1 запрос в секунду на пользователя
6. **Алерты при перерасходе** — Telegram директору если бюджет > 80%

### 7.1 Уровни доступа к AI Knowledge Documents

```yaml
access_levels:
  Public:
    visible_to: All authenticated users
    examples: ["Шаблоны договоров", "Технические инструкции"]

  Internal:
    visible_to: All employees
    examples: ["Активные проекты", "Сметы (без цен ниже)", "Графики работ"]

  Confidential:
    visible_to: Director, ПТО, Главный инженер
    examples: ["Финансы проектов", "Маржи", "Закупочные цены"]

  Director Only:
    visible_to: Director only
    examples: ["Зарплаты", "Отношения с заказчиками", "Стратегические вопросы"]
```

### 7.2 Фильтрация при поиске

```python
def search_with_acl(query, user):
    user_levels = user.allowed_access_levels()
    # ["Public", "Internal", "Confidential"] для ПТО

    results = qdrant.search(
        query_vector=embed(query),
        filter={"access_level": {"$in": user_levels}},
        limit=20
    )
    return results

# Если ПТО спросит про зарплаты → просто не найдёт документов
# AI ответит: "Нет данных по этому вопросу"
```

## 8. Интерфейс

### 8.1 Веб-интерфейс (10-й кастомный экран)

Похож на ChatGPT, но в стиле «Олимп ERP» (Apple-style):

- **Левая панель:** список conversations (последние + закреплённые)
- **Центр:** текущий чат со streaming-ответами
- **Правая панель** *(опционально)*: контекст текущей беседы, источники

**Особенности:**
- Возможность прикрепить файл (PDF/Excel) — AI проанализирует
- Голосовой ввод (микрофон → Whisper)
- Кнопки suggested actions внизу ответа
- Citations в виде кликабельных ссылок на исходные документы

### 8.2 В Telegram-боте (для прорабов и Димы на ходу)

```
Команды:
/ai <вопрос>    — задать вопрос текстом
голосовое       — задать вопрос голосом

Особенности:
- Краткие ответы (TG ограничения по длине)
- Inline-кнопки для actions
- Возможность продолжить в веб-чате (deep link)
```

### 8.3 Виджет на дашборде директора

В правом верхнем углу:
- Кнопка «Спросить AI»
- При клике → раскрывается мини-чат
- Pre-filled подсказки: «Что мне сегодня делать?», «Где риски?», «Какие тендеры подавать?»

## 9. Метрики успеха

### 9.1 Технические

| Метрика | Цель |
|---------|------|
| Время ответа (p95) | < 8 секунд |

| Стоимость на ответ (среднее) | < 5 ₽ |

| Метрика | Цель |

| Принято решений на основе AI-инсайтов | > 5 в неделю |
| Полезность по фидбэку | > 75% «полезно» |

- **Хранение разговоров:** 2 года, с правом удаления по запросу

## 13. Альтернативные подходы (если что-то не пойдёт)

### Если LLM-стоимость слишком высокая
→ Self-hosted Llama 3 70B на собственном GPU-сервере. Инвестиция ~500K ₽ в железо, но на горизонте 2 лет — окупается при текущем потреблении

### Если RAG не даёт точности
→ Гибрид: RAG + structured queries напрямую в БД через MCP-протокол (Claude умеет работать с БД)

### Если директор не использует
→ Делаем всё через Telegram (привычный канал) и встраиваем напоминания в дашборд

### Если данные слишком разрозненные для индексации
→ Этап 1 — только тендеры и сметы (структурированные); потом постепенно
