## 0. Зачем это нужно

«Олимп» использует строительную технику и оборудование. Сейчас всё ведётся:
- В голове Димы и главного инженера
- В Excel-файлах
- В чеках на ГСМ, которые приносит прораб
- В разговорах: «А когда последний раз делали ТО на подъёмнике?»

**Проблемы:**
1. **Простой техники** — техника стоит без работы, а её можно было бы использовать на другом объекте или сдать в аренду
2. **Неучтённые расходы ГСМ** — прорабы заправляют, а потом непонятно куда списать
3. **Просроченные ТО / страховки / поверки** — узнаём когда уже поздно
4. **Не знаем стоимости владения** — а это ключевая цифра для решения «купить vs арендовать»
5. **Хищения** — тяжёлая для отслеживания тема, но реальная

**Цель модуля:** Дима за минуту видит:
- Где какая техника
- Когда следующее ТО
- Сколько ГСМ съели за месяц
- Какая техника простаивает > 7 дней

## 1. Что делаем (MVP, Фаза 8)

### 1.1 Учёт техники

```
Каждая единица техники = карточка Equipment с:
  - инвентарный код, наименование, тип, год выпуска
  - VIN/госномер (если транспорт)
  - покупная стоимость, текущая балансовая
  - статус: Доступна / На объекте / На ТО / В ремонте / Списана
  - текущее местоположение, ответственный
  - даты: следующее ТО, страховка, поверка, СРО
  - моточасы / пробег
  - фото и документы
```

### 1.2 Журнал ТО (Maintenance Log)

```
Каждое ТО / ремонт = запись с:
  - дата, тип (плановое/внеплановое/ремонт/поверка)
  - что сделали (описание)
  - кто делал (своя ремонтная или сторонняя)
  - стоимость работ + запчасти = итого
  - акт + фото
  - что обновляется в Equipment: следующее ТО
```

### 1.3 Журнал ГСМ (Fuel Log)

```
Каждая заправка = запись с:
  - дата, время, заправка, литры, цена, сумма
  - кто заправлял (прораб, водитель)
  - на какой проект списать
  - GPS-координаты (для контроля)
  - чек (фото)
  - показания одометра / моточасов
```

## 2. Что НЕ делаем (вне MVP)

- ❌ GPS-трекинг в реальном времени — это ~200 тыс ₽ на устройства + абонентка
- ❌ Telematics-интеграция (CAN-шина грузовиков)
- ❌ Биометрия / RFID на водителе
- ❌ Сложная аналитика износа (по типу telemetry-as-a-service)
- ❌ Автоматическая связка с топливными картами (Лукойл/Газпромнефть API)
- ❌ Аренда техники сторонним компаниям (модуль аренды)

Всё это можно добавить позже, **если простые механизмы покажут ценность**.

## 3. Типы техники в «Олимпе»

Учитывая профиль работ:

### 3.1 Подъёмная техника
- **Ножничные подъёмники** (Genie GS, Haulotte) — для АКЗ резервуаров, кровли
- **Телескопические подъёмники** (Genie Z, JLG) — для фасадов, мостов
- **Башенные краны** (свои или арендованные)
- **Автокраны** (Liebherr, Tadano)

### 3.2 Окрасочное и АКЗ оборудование
- Безвоздушные окрасочные аппараты (Graco, WAGNER)
- Краскопульты, шланги, наконечники
- Сушки, обогреватели для окраски
- Дозирующие насосы для двухкомпонентных красок

### 3.3 Бетонные работы
- Бетономешалки (на объекте)
- Вибраторы глубинные
- Виброплиты, виброрейки
- Заглаживающие машины

### 3.4 Сварочное оборудование
- Сварочные аппараты (полуавтомат, инвертор, ТИГ)
- Сварочные инверторы, выпрямители
- Резаки

### 3.5 Промальп оборудование (важная категория!)
- Верёвки статические (статика 11мм)
- Спусковые устройства (рапид, петцл, стоп)
- Зажимы (жумары, кролы)
- Обвязки и страховки
- Каски
- Большой вопрос: **сертификация и поверка** — раз в 12 месяцев обязательно!

### 3.6 Прочее
- Компрессоры
- Генераторы
- Перфораторы, отбойные молотки
- Тепловые пушки
- Леса (свои и арендованные)
- Грузовая техника (самосвал, манипулятор)

## 4. Жизненный цикл единицы техники

```
[Куплена] → [Введена в работу] ──┐
                                 ↓
                            [Доступна]
                                 ↓
                    ┌─────[На объекте]─────┐
                    ↓                       ↓
              [Возврат]                [На ТО / Ремонт]
                    ↓                       ↓
              [Доступна]              [Доступна]
                                            
                                      ↓
                                 [Списана / Продана]
```

### Workflow в системе

```yaml
DocType Equipment, поле status:
  - "Доступна" — лежит на складе или у ответственного
  - "На объекте" — в работе на конкретном проекте
  - "На ТО" — у механика
  - "В ремонте" — внеплановый ремонт
  - "Сдана в аренду" — отдали стороннему
  - "Списана" — выведена из эксплуатации
```

При смене статуса:
- "На объекте" → требует Project + Responsible
- "На ТО" → автоматически создаётся черновик Maintenance Log
- "Списана" → переход в архив, доступ только для отчётов

## 5. Расчёт стоимости владения

Самая полезная цифра для директора. Помогает решать «продать vs ремонтировать».

```python
def calculate_total_cost_of_ownership(equipment, period_months=12):
    """Стоимость владения за период"""

    # 1. Амортизация
    depreciation = equipment.purchase_price * (equipment.depreciation_rate_pct / 100) * (period_months / 12)

    # 2. Расходы на ТО и ремонты
    maintenance_costs = sum(
        log.total_cost
        for log in equipment.maintenance_logs
        if log.log_date in last_period(period_months)
    )

    # 3. ГСМ (если применимо)
    fuel_costs = sum(
        log.total_amount
        for log in equipment.fuel_logs
        if log.log_date in last_period(period_months)
    )

    # 4. Страховка
    insurance = equipment.annual_insurance * (period_months / 12)

    # 5. Транспортный налог (для самоходной)
    tax = equipment.annual_tax * (period_months / 12)

    # 6. Время простоя (упущенная прибыль)
    idle_days = count_days_with_status(equipment, "Доступна", period_months)
    if equipment.daily_rent_rate:  # сколько мог зарабатывать
        idle_loss = idle_days * equipment.daily_rent_rate * 0.6  # 60% средняя загрузка

    total = depreciation + maintenance_costs + fuel_costs + insurance + tax

    return {
        "total": total,
        "breakdown": {...},
        "per_engine_hour": total / equipment.engine_hours_in_period,
        "per_workday": total / equipment.workdays_in_period,
        "missed_revenue": idle_loss
    }
```

### Пример отчёта

> **Подъёмник Genie Z-45/22 (EQ-0014)**
> Стоимость владения за 2025: **485 200 ₽**
> Из них:
> - Амортизация: 240 000 ₽ (10% от 2,4 млн)
> - ТО и ремонты: 87 600 ₽ (3 ТО + 1 ремонт пневмосистемы)
> - ГСМ: 124 000 ₽ (1 240 л дизеля)
> - Страховка: 28 600 ₽
> - Налог: 5 000 ₽
>
> Использовался 184 рабочих дня. Стоимость 1 рабочего дня: **2 637 ₽**.
> На 7 проектах. Самый эффективный: Marins Park (74 дня).
>
> ⚠️ Простой 91 день. Упущенная прибыль (если бы сдали в аренду): ~210 000 ₽.

## 6. Алерты и уведомления

### 6.1 Авто-алерты в Telegram

```yaml
- За 30 дней до ТО: уведомление гл.инженеру
- За 7 дней до ТО: уведомление директору
- ТО просрочено: блокировка статуса "На объекте"

- За 60 дней до окончания страховки: уведомление
- За 30 дней до поверки (краны, подъёмники): уведомление
- За 30 дней до СРО: уведомление

- Аномальный расход ГСМ (>30% от нормы): алерт
- Простой техники >7 дней: алерт «можно сдать в аренду»
- ГСМ без чека: алерт
- ГСМ заправлен в нерабочее время: алерт
```

### 6.2 На дашборде

Виджет «Техника» в командном центре:
- Сводка: 12 единиц / 8 в работе / 3 на ТО / 1 простаивает
- Алерты: ТО на этой неделе (2), страховки в этом месяце (1)

## 7. Интеграции

### 7.1 С Project и Foreman Report

Прораб в Telegram-боте:
```
/equipment list — список техники на моём объекте
/equipment fuel <литры> — записать заправку
/equipment issue <описание> — сообщить о проблеме
```

При создании Foreman Report:
- Поле «Техника на объекте» с галочками
- Если галочка стоит: пометка дня работы для расчёта моточасов

### 7.2 С Stock и Purchase Order

При покупке запчасти:
- В PO ставится галочка «Это для техники EQ-XXXX»
- Автоматически связывается с Maintenance Log

### 7.3 С финансами

ГСМ списывается на конкретный проект (Cost Center):
```
Fuel Log → Journal Entry автоматически:
  Dr: Cost Center проекта
  Cr: Касса / Подотчёт прораба
```

## 10. Метрики успеха модуля

| Метрика | Цель |
|---------|------|
| % просроченных ТО | < 5% (было ~30%) |
| Время простоя техники | < 10% времени |
| Расход ГСМ зафиксирован | 100% (было ~70%) |
| Перерасход ГСМ обнаружен | 100% случаев |
| Время «найти технику и узнать её состояние» | < 1 минуты |
| Стоимость владения известна | для каждой единицы |
| Решений «продать/обновить» на основе данных | > 80% |

## 12. Что нужно решить директору ДО запуска

1. **Полный список техники** — Excel-шаблон должен заполнить главный инженер
2. **Лимиты ответственности** — кто за что отвечает
3. **Правила заправки** — где можно/нельзя, какие чеки нужны
4. **Стоимость аренды каждой единицы** — для расчёта TCO и упущенной прибыли
5. **Тарифы амортизации** — стандартные 10% или другие
