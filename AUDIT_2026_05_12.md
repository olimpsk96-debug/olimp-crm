## Критичные баги (требуют немедленного фикса)

### 1. TenderLaw несоответствие
- **Frontend** (`types/tender.ts:10`): `"44-ФЗ" | "223-ФЗ" | "Прямой договор"`
- **Backend** (`api/tender.py:39`): возвращает `"Коммерческий"`
- **Эффект**: AI-оценка не работает для коммерческих тендеров

### 2. Дублирование STATUSES_ACTIVE
- В 3 файлах одинаковая константа: `telegram_utils.py`, `api/dashboard.py:30`, `api/crm.py:26`
- Должна быть одна — в `telegram_utils.py` (или новом `constants.py`)

### 3. Supply статус «Получено» vs «Получена»
- В одном месте мужской род, в другом женский. UI ломается при фильтрации.

### 4. ProjectDetail.estimate (уже починен в API)
- В `api/project.py` используется `getattr(est, "base_total", 0)` — правильно
- Но в `types/project.ts:71` тип всё ещё ожидает `total_cost`/`total_price` — переименовать в `base_total`/`our_total`

### 5. Безопасность: cash_balance в `frappe.db.get_default()`
- Это глобальный default — небезопасное хранилище для финансовых данных
- Должно быть в своём DocType `Olimp Settings`
