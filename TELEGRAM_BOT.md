# TELEGRAM_BOT.md — Спецификация Telegram-бота

**Назначение:** Telegram — основной канал общения системы с людьми вне офиса (прорабы, директор в дороге).

**Технология:** n8n workflows + Telegram Bot API + кастомный Python-сервис для распознавания (Whisper API).

**Принцип:** простота. Прораб не должен изучать команды — голосом и фото всё работает.

---

## 1. Боты в системе

В Telegram у нас будет **один бот** с разными ролями для разных пользователей:

```
@OlimpErpBot
├── Роль "Директор" (Дима) — алерты, согласования, быстрые отчёты
├── Роль "Прораб" — отчёты, фото, проблемы
├── Роль "ПТО / Сметчик" — нотификации по тендерам, КС-2
└── Роль "Бухгалтер" — реестр платежей, поступлений
```

Роли определяются по `chat_id` пользователя в Frappe (поле `telegram_chat_id` в Employee/User).

## 2. Сценарии для ДИРЕКТОРА (Дима)

### 2.1 Утренний брифинг

Каждое утро в 08:00 (по будням) бот отправляет:

```
🌅 Доброе утро, Дмитрий!

📊 Командный центр на 09.05:

💰 Выручка YTD:        52,8 млн ₽ (+12,4%)
📈 Маржа портфеля:     37,2% ↗
💸 Кассовый разрыв:    через 7 дней (−2,1 млн)

🔥 Сегодня требуют решения:
• 3 тендера: дедлайн < 3 дней
• 1 КС-2 на согласовании (Marins, 2,4 млн)
• 3 истекающих допуска ОТ/ТБ

🎯 Приоритет дня:
ТЭЦ-3 АКЗ — дедлайн 15.05 (1,8 дня), смета на проверке

[Открыть дашборд]   [Подробнее]
```

### 2.2 Алерты в реальном времени

| Триггер | Сообщение |
|---------|-----------|
| Новый тендер AI score > 70% | `🎯 Новый тендер: <название>, НМЦ X млн, AI рекомендует` |
| Дедлайн тендера < 24ч | `⚠️ Дедлайн через X часов: <тендер>. Готовность 3/5` |
| КС-2 на согласовании | `📄 КС-2 на согласование: <проект>, X млн ₽ [Согласовать] [Отклонить]` |
| Поступление > 500к | `💰 Получено X млн ₽ от <заказчик> по <проект>` |
| Кассовый разрыв через 7д | `⚠️ Прогноз кассового разрыва через 7 дней: −X млн` |
| Ошибка маржи проекта | `📉 Маржа <проект> упала ниже 25% (была 35%)` |
| Допуск ОТ/ТБ < 7 дней | `🦺 У <ФИО>: допуск <тип> истекает через X дней` |

### 2.3 Команды

```
/start              - регистрация / приветствие
/dashboard          - метрики прямо в чате
/tenders            - 3 горячих тендера
/projects           - 7 активных, маржа, готовность
/cash               - cashflow на 30 дней
/today              - что требует решения сегодня
/menu               - inline-меню всех функций
```

### 2.4 Inline-кнопки на согласование КС-2

```
📄 КС-2 №3 для согласования

Проект:      Кровля Marins Park Hotel
Этап:        3 (Утеплитель и стяжка)
Сумма:       2 100 000 ₽
Период:      02.05 - 12.05.2026

Подготовил:  Сидоров А.Н. (ПТО)

[👁 Посмотреть PDF]
[✅ Согласовать]  [❌ Отклонить]
[💬 Комментарий]
```

При нажатии "Согласовать" — статус КС-2 → "Подписан заказчиком" (если есть скан) или → "На подписи у заказчика".

## 3. Сценарии для ПРОРАБА

### 3.1 Утренний пинг

В 07:30 (по будням) бот отправляет:

```
☀️ Доброе утро, Иванов А.С.!

📍 Сегодня: Marins Park (объект PR-2026-04-12)

✅ Команда (плановая):     5 человек
📋 План на сегодня:        Этап 3 — стяжка
🌡️ Погода:                 +18°C, без осадков
🦺 Внимание:               у тебя истекает допуск 12.05

[📸 Загрузить фото] [📝 Отчёт] [⚠️ Проблема]
```

### 3.2 Команды

```
/start              - регистрация (привязка к Employee)
/today              - твой объект сегодня
/team               - кто на смене (по объекту)
/photo              - быстрая загрузка фото с привязкой
/report             - отчёт за день (текст или голос)
/issue              - сообщить о проблеме
/material           - запросить материал
/ks                 - текущий статус закрытия по этапам
```

### 3.3 Загрузка фото (главный сценарий)

Прораб просто **отправляет фото в чат**. Бот определяет объект автоматически:

```
Прораб: [фото]

Бот: 📸 Получил! Это для Marins Park, верно?

[✅ Да, сохранить]  [Другой объект]  [Отмена]
```

После выбора:
- Фото сохраняется в MinIO
- Создаётся `Photo Report` запись
- Привязывается к Project, дате, прорабу
- Доступно директору и ПТО в системе

**Если несколько фото подряд** — группируются в один отчёт.

### 3.4 Голосовой отчёт

Прораб пишет голосовое сообщение → Whisper API распознаёт → создаётся `Foreman Report`.

```
Прораб: [голос: "Сегодня закончили утеплитель на восточном скате,
            сделали 60% стяжки, завтра доделаем. Бригада в полном составе.
            Привезли два рулона рубероида, не хватает один."]

Бот: ✅ Записал отчёт за 09.05.

📝 Распознано:
"Сегодня закончили утеплитель на восточном скате, сделали 60% стяжки,
завтра доделаем. Бригада в полном составе. Привезли два рулона рубероида,
не хватает один."

🔍 Что я заметил:
• Прогресс этапа: ~75% (по упоминаниям)
• Запрос материала: 1 рулон рубероида

Создать заявку на материал? [Да] [Нет] [Изменить отчёт]
```

### 3.5 Сообщение о проблеме

```
Прораб: /issue
Бот: 🚨 Что случилось? Опишите коротко (текст или голос)

Прораб: "Заказчик не пускает на крышу — ругаются что мусор оставили
         после этапа 2. Нужно срочно убрать или объяснить."

Бот: ⚠️ Записал. Уведомил Дмитрия и ПТО.
     Тип: конфликт с заказчиком
     Срочность: высокая

[📞 Позвонить директору]
```

Уведомление директору приходит мгновенно с inline-кнопкой "Решено / В работе".

## 5. Архитектура реализации

### 5.1 Стек

```
[Telegram Bot API]
        ↕  HTTPS webhooks
[n8n workflows]  ←→  [Frappe ERPNext API]
        ↕                     ↕
[Whisper API]            [MinIO (фото)]
[Claude API]
```

### 5.2 n8n workflows (4 главных)

1. **`telegram_router.json`** — главный entry-point, разводит сообщения по типам:
   - текст с командой `/...` → command handler
   - текст без команды → AI-обработка контекста
   - фото → photo handler
   - голос → voice handler
   - inline-кнопка callback → action handler

2. **`telegram_morning_brief.json`** — cron 08:00 будни → отправка брифинга директору и прорабам

3. **`telegram_alerts.json`** — слушает webhooks из Frappe, отправляет нотификации

4. **`telegram_voice_to_report.json`** — обработка голоса:
   - скачать file_id из Telegram
   - отправить в Whisper API
   - извлечь сущности (Claude API)
   - создать `Foreman Report` через Frappe API

### 5.3 Регистрация пользователя

Первый запуск:

```
Прораб: /start
Бот:   👋 Привет! Напишите ваш номер телефона для привязки.

Прораб: +79991234567
Бот:   🔍 Ищу...
       ✅ Иванов А.С., прораб, объект Marins Park.
       Привязка готова. Завтра в 07:30 пришлю утренний пинг.
```

Если не найден — фолбэк на ручную привязку через Диму.

### 5.4 Безопасность

- Бот публичный, но без `chat_id` в `Employee.telegram_chat_id` команды не работают
- `/start` требует подтверждения по телефону
- Все действия логируются (frappe.log_error для debug)
- Inline-кнопки на согласование КС-2 проверяют permissions

## 7. Что не делаем в боте

❌ Полноценное ведение проектов (для этого Next.js)
❌ Сложные таблицы, графики (только summary)
❌ Создание тендеров с нуля (только просмотр)
❌ Работа со сметой (Гранд-Смета осталась)
❌ Заполнение объёмных форм

Бот — это **уведомления + быстрые действия + полевой ввод**, не замена основного UI.

---

## 8. Метрики успеха

- 100% прорабов используют бот ежедневно (не Excel, не WhatsApp)
- Время от инцидента на стройке → уведомление директора: < 5 минут
- Время согласования КС-2 (Дима получил → решение): < 30 минут (было часы / дни)
- Голосовых отчётов в день: > 5 (нет психологического барьера)
- Фото в системе по проекту: > 20 за неделю работы
