# Дорожная карта внедрения OLIMP.STACK

> Срок: **6 недель** до первой реальной сметы в OCE
> **3 месяца** до полной интеграции с ERPNext
> Команда: ты + Claude Code (десктоп)

## Фаза 0. Подготовка (1 неделя)

### Цель
Понять, что технически готово, а что нужно доустановить.

### Задачи

- [ ] **Аудит текущего docker-compose ERPNext-стека**
  - Проверь, что `olimp_network` объявлена как external и доступна другим compose-файлам
  - Если нет — выдели сеть, привяжи к ней все существующие сервисы
- [ ] **Запасти ресурсы на VPS Beget**
  - OCE требует: 2 ГБ RAM, 1 vCPU, 20 ГБ SSD (минимум)
  - Postgres OCE: +1 ГБ RAM
  - LanceDB embedded: +500 МБ RAM (без vector DB вынесения)
  - Итого: убедись, что у VPS есть запас в 4 ГБ RAM сверх текущих расходов ERPNext
- [ ] **Завести поддомен** `estimate.olimp-ural.ru` в DNS, направить на VPS
- [ ] **Сгенерировать секреты** для `.env.olimp` (см. `.env.olimp.example`)
  - `openssl rand -base64 48` для каждого пароля
- [ ] **Купить API-ключ OpenAI** (если ещё нет) — для embeddings
  - Можно обойтись без OpenAI, заменив на self-hosted bge-m3, но дольше настройка
- [ ] **Сделать бэкап** текущего ERPNext перед любыми изменениями
  - `bench --site all backup` + копия `sites/` в MinIO

### Проверка готовности
```bash
docker network inspect olimp_network              # сеть есть и external
free -h                                            # >4 ГБ свободно
ping estimate.olimp-ural.ru                        # DNS резолвится
cat .env.olimp | grep -c "ChangeMe"               # должно быть 0 (все заменены)
ls -la sites/*/private/backups/ | tail -3         # свежие бэкапы есть
```

## Фаза 1. Развёртывание OCE (1 неделя)

### Цель
Поднять OCE рядом с ERPNext, без интеграции, для теста.

### Задачи

- [ ] **Клонировать репозиторий OCE**
  ```bash
  cd /opt/olimp
  git clone https://github.com/datadrivenconstruction/OpenConstructionERP.git vendor/oce
  cd vendor/oce && git checkout v2.0.0   # последний стабильный тег
  ```
- [ ] **Положить наши файлы рядом**
  ```bash
  cp docker-compose.olimp.yml /opt/olimp/
  cp .env.olimp /opt/olimp/
  cp -r scripts/ /opt/olimp/
  ```
- [ ] **Запустить OCE-стек**
  ```bash
  cd /opt/olimp
  docker compose -f docker-compose.yml -f docker-compose.olimp.yml up -d
  docker compose logs -f oce-backend   # ждём "Application startup complete"
  ```
- [ ] **Открыть** `https://estimate.olimp-ural.ru`
  - Залогиниться демо-аккаунтом
  - Создать тестовый Project "TEST-NTMK"
  - Прокликать BOQ-редактор, проверить, что работает
- [ ] **Локализация на русский**
  - В UI Settings: Language → Russian
  - Region → Russia
  - Currency → RUB
  - Standard → ГЭСН
- [ ] **Залить лого Олимпа**
  - SVG-лого в `/opt/olimp/oce-custom/branding/logo-olimp.svg`
  - Перезапустить `oce-frontend`

### Проверка
- [ ] Откатывается из бэкапа за 5 минут
- [ ] Прометей собирает метрики `oce-backend` (если есть мониторинг)

## Фаза 4. Интеграция с ERPNext (2 недели)

### Цель
Связать OCE и ERPNext через n8n, чтобы выигранные сметы превращались в проекты.

### Задачи (n8n workflows)

- [ ] **Workflow 1: BOQ → Project в ERPNext**
  - Триггер: POST на `/webhook/boq-to-erpnext` (вызывается из OCE при смене статуса BOQ → Won)
  - Действия:
    1. Получить полный BOQ из OCE API
    2. Создать Project в ERPNext с правильным бюджетом
    3. Для каждой секции BOQ — создать Task
    4. Для каждой позиции с типом "материал" — создать Material Request
    5. Записать `custom_oce_boq_id` обратно в Project
  - Telegram-уведомление: "Проект НТМК-Фундамент создан в ERPNext, бюджет 28 млн ₽"

- [ ] **Workflow 2: Факт расходов → EVM**
  - Триггер: каждую ночь в 23:00 (cron)
  - Действия:
    1. По всем активным Project в ERPNext — собрать факт (закупки, ЗП, КС-2 от субов)
    2. POST в OCE `/api/projects/{id}/actuals`
    3. OCE пересчитывает SPI, CPI, EAC
    4. Если EAC > бюджет × 1.05 → Telegram директору

- [ ] **Workflow 3: Tender — рассылка субам**
  - Триггер: при создании Tender в OCE
  - Действия:
    1. Из OCE: получить позиции и список потенциальных субов
    2. В ERPNext: создать Request for Quotation с прикреплёнными позициями
    3. Subcontractors получают email через Supplier Portal
    4. Когда все Supplier Quotation поступили — собрать сравнительную таблицу,
       отправить обратно в OCE для side-by-side анализа

### Custom Fields в ERPNext
- [ ] Добавить через UI: Customize Form → Project → +8 полей (см. mapping doc)
- [ ] Добавить: Customize Form → Task → +7 полей
- [ ] Создать новые DocType:
  - [ ] BOQ + BOQ Section (child) + BOQ Position (child)
  - [ ] Construction Assembly + Assembly Item (child)
  - [ ] Project Change Order
  - [ ] Project Risk

### Проверка
- [ ] От Won в OCE до Project в ERPNext проходит <30 сек
- [ ] EVM-метрики появляются на дашборде директора в ERPNext
- [ ] Telegram-алерты приходят в нужный чат

## Бюджет проекта (приблизительный)

| Статья | Стоимость | Периодичность |
|--------|-----------|--------------|
| VPS Beget (расширение под OCE) | +1500 ₽/мес | ежемесячно |
| OpenAI API (embeddings) | ~$10/мес при 5000 позиций | ежемесячно |
| Anthropic API (Claude для AI Cost Advisor) | $30-100/мес | ежемесячно |
| Домен estimate.olimp-ural.ru | 200 ₽ (уже есть) | ежегодно |
| Твоё время на внедрение | ~80-100 часов | разово |
| **Итого CapEx** | **~100 часов** | разово |
| **Итого OpEx** | **~4000-12000 ₽/мес** | ежемесячно |

ROI: при сохранении ~25 часов в неделю на сметах = 100 часов в месяц.
При оценке твоего времени как директора в 3000 ₽/час → 300 тыс ₽/мес экономии.

## Критерии остановки (когда бить тревогу)

- **Фаза 1 (развёртывание):** если за 3 дня не поднялось — откатываемся,
  переписываем модули BOQ как DocType в ERPNext (вариант B из обсуждения)
- **Фаза 2 (данные):** если AI Cost Advisor выдаёт >50% некорректных матчей —
  значит, данные не дозалили или embeddings не подходят, нужна доработка
- **Фаза 3 (первая смета):** если время >3 часов и расхождение >15% —
  что-то не настроено в Cost Database, возвращаемся к Фазе 2
- **Фаза 4 (интеграция):** если n8n workflows не стабильны (падают чаще 1/100)
  — переписываем как Python-микросервис, n8n оставляем только для триггеров
