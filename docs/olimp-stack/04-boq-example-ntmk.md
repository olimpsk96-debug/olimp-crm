# Пример BOQ в OCE: Фундамент 7×7×2м на ЕВРАЗ НТМК

> Этот документ показывает, **как именно** OCE заменил бы твою текущую работу
> с Excel + Гранд-Сметой на реальном объекте, который ты делаешь сейчас.

## Объект

- **Заказчик:** ЕВРАЗ НТМК (Нижний Тагил)
- **Объём:** Фундаментная плита 7×7×2 м с анкерными группами под технологическое
  оборудование
- **Текущая работа:** Две вариации нагрузок, целевая стоимость 26–36 млн ₽ с НДС
- **Сейчас делаешь:** Excel + Гранд-Смета + ручной расчёт ресурсов

## Что меняется с OCE

### Шаг 1. Загрузка ТЗ и РД (5 минут)

Сейчас: ты читаешь PDF от заказчика, выписываешь объёмы вручную, вбиваешь в Гранд-Смету.

**В OCE:**
```
1. Создаёшь Project "НТМК-2026-Фундамент-7x7"
2. Загружаешь PDF/DWG в раздел Documents
3. Открываешь PDF в браузерном просмотрщике
4. Калибруешь масштаб (клик-клик по размерной линии)
5. Кликами обводишь контур плиты → автоматически считается площадь 49 м²
6. Кликами выделяешь зону армирования → объём бетона 98 м³, сетка 30 кг/м³
7. Кликаешь на анкерные группы → counter говорит "32 шт. М30×600"
```

Результат: за 5 минут у тебя точные количества с привязкой к чертежу.
Если заказчик пришлёт ревизию РД — масштаб уже откалиброван, перемеряешь за минуту.

### Шаг 2. AI Cost Advisor генерирует драфт BOQ (30 секунд)

Сейчас: ты вручную выписываешь позиции в Гранд-Смету, ищешь расценки ГЭСН.

**В OCE:**
```
Промпт: "Сгенерируй BOQ для фундаментной плиты 7×7×2м из бетона В25
с двойным армированием 30 кг/м³ и 32 анкерами М30 длиной 600мм.
Используй ГЭСН-2020 + рыночные расценки УрФО."

Через 30 секунд:
✅ Раздел 1. Подготовительные работы
   1.1. Снятие плодородного слоя h=15см, S=80м²
   1.2. Разработка котлована, V=120 м³
   1.3. Уплотнение основания
   1.4. Устройство подбетонки В7.5, V=4 м³

✅ Раздел 2. Армирование
   2.1. Арматура А500С Ø16, m=2.94 т
   2.2. Арматура А500С Ø10 (хомуты), m=0.5 т
   2.3. Установка закладных деталей
   2.4. Монтаж анкерных групп М30×600, n=32 шт

✅ Раздел 3. Бетонирование
   3.1. Опалубка щитовая, S=84 м²
   3.2. Бетон В25 F150 W6, V=98 м³
   3.3. Уход за бетоном

✅ Раздел 4. Сопутствующие
   4.1. Геодезическая разбивка
   4.2. Контроль качества бетона (7 серий)
   4.3. Исполнительная съёмка
```

Этот драфт ты правишь — не пишешь с нуля.

### Шаг 3. Match на твою базу расценок (1 минута)

Сейчас: каждую позицию из ГЭСН ты вручную «подкручиваешь» под рынок УрФО.

**В OCE:**
- AI сматчил позиции на расценки из CWICR (база OCE)
- Для каждой позиции ты видишь: ГЭСН (с индексом Минстроя), CWICR-рынок, твоя историческая цена с прошлых объектов
- Кнопка «Применить ко всем» — подставляет твою историческую цену
- Подсвечивает позиции, где разброс >20% — это где нужно подумать

### Шаг 4. Ресурсная разбивка и сборки (2 минуты)

Сейчас: трудозатраты и материалы — отдельная таблица, считаешь в калькуляторе.

**В OCE:**
- Каждая позиция автоматически разбита на ресурсы из Assembly
- Например, "Бетон В25 в опалубке, 1 м³" = сборка:
  - 1 м³ бетон В25 (1100 ₽/м³)
  - 0.05 чел-смена бетонщика 5р (4500 ₽/смена)
  - 0.02 машино-смены крана (12000 ₽/смена)
  - 0.5 кг ПВЛ (45 ₽/кг)
- Изменил цену бетона у одного поставщика — пересчиталась вся смета

### Шаг 5. Наценки и итог (10 секунд)

```
Сумма прямых затрат: 18 500 000 ₽
+ Накладные расходы 8%: 1 480 000 ₽
+ Плановая прибыль 15%: 2 997 000 ₽
+ Резерв на риски 5%: 1 148 850 ₽
─────────────────────────────────
Итого без НДС: 24 125 850 ₽
+ НДС 20%: 4 825 170 ₽
═════════════════════════════════
ВСЕГО с НДС: 28 951 020 ₽
```

Видишь сразу две версии (минимальная нагрузка / максимальная нагрузка) в одном окне.

### Шаг 6. Валидация (5 секунд)

OCE прогоняет 42 правила, типа:
- ⚠️ Позиция 2.4 (анкеры): объём в шт, но total указан за тонну — проверь
- ⚠️ Позиция 3.2 (бетон): марка В25 F150 W6, но в спецификации к чертежу В30 F200
- ✅ Все ГЭСН-коды актуальные (база на май 2026)
- ✅ Сумма секций сходится с grand_total

### Шаг 7. Экспорт (20 секунд)

```
Формат → Сразу несколько:
  📄 КП в PDF (для заказчика, с шапкой Олимпа)
  📊 Excel-расшифровка (для коммерческой службы НТМК)
  📋 Гранд-Смета формат (если попросят официалку)
  💾 JSON в n8n для отправки в Twenty CRM
```

### Шаг 8. Версионирование

Заказчик прислал замечания → правишь BOQ → создаётся v2.0.
В истории видишь, что изменилось между v1.0 и v2.0 (как git diff).

### Шаг 9. Тендер выиграли — отправка в ERPNext

Один клик:
```
Project "НТМК-2026-Фундамент" создаётся в ERPNext с:
  - Бюджет 28 951 020 ₽
  - 18 задач из секций BOQ с датами
  - 42 Material Request (под закупки бетона, арматуры, опалубки)
  - 3 Subcontract Request (подбетонка, арматурные работы, бетонирование)
```

Прораб открывает ERPNext, видит свой объект, начинает работать.

## Прямой выигрыш по времени

| Этап | Сейчас (Excel + Гранд) | В OCE | Экономия |
|------|----------------------|-------|----------|
| Чтение РД и обмеры | 2-3 часа | 10 минут | ~150 мин |
| Создание позиций | 1-2 часа | 5 минут (драфт от AI) | ~100 мин |
| Расценки и торговля | 2-3 часа | 30 минут | ~120 мин |
| Ресурсная разбивка | 1 час | автоматически | ~60 мин |
| Наценки и итог | 30 минут | мгновенно | ~30 мин |
| Валидация | вручную, могут быть пропуски | 5 секунд | надёжность |
| Экспорт КП | 30 минут | 20 секунд | ~30 мин |
| **Итого на одну смету** | **~8 часов** | **~45 минут** | **~7 часов** |

При 3-4 сметах в неделю — это **20–25 часов в неделю**, которые ты возвращаешь
себе или сметчику. Полтора рабочих дня в неделю на руководителя или 0.5 ставки сметчика.

## Прямой выигрыш по качеству

1. **Меньше ошибок:** валидация ловит то, что глаз пропускает (несоответствие
   марки бетона, забытые позиции, неверные единицы).
2. **Базовая память:** твои цены с предыдущих объектов автоматически
   подставляются — не теряешь ноу-хау при увольнении сметчика.
3. **Защита перед заказчиком:** есть Monte Carlo по рискам → можешь
   обосновать `почему 28, а не 26`.
4. **Версионирование:** не теряются правки, видна история переговоров.

## Прямой выигрыш для управления

После выигрыша тендера, в ERPNext автоматически:
- Заведён проект с правильным бюджетом
- Создан график закупок (бетон надо за 2 недели, арматура за неделю)
- Закреплены прорабы и сроки
- Запущен EVM-мониторинг: каждый день видишь SPI/CPI прямо в Telegram

Если на 30% готовности объекта CPI < 0.9 — у тебя 70% работ осталось,
чтобы успеть исправиться. Сейчас ты узнаёшь о перерасходе по факту КС-2 в конце месяца.

## Как это масштабируется на другие объекты в работе

- **Антресоли Волгоградская:** PDF-такеофф особенно ценен — там сложные узлы крепления
- **Внутренние сети ВК Прософт-Системы:** 133 позиции — без AI ты их вручную набивал
- **СВЕЗА ФК-2:** реконструкция, рисков много → Monte Carlo критичен
- **PP-R RUBIS Ø63:** маленький объект, шаблонная Assembly «PP-R под ключ»
  один раз создаётся → используется всегда
